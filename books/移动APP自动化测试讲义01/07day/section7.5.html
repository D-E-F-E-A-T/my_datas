Git

目标

1. 了解GitHub
2. 掌握Git常用命令

一、Git介绍

1.1 下载

https://gitforwindows.org/

# 安装完毕，进行测试
git version

1.2 配置

$ git config --global user.name "John Doe"
$ git config --global user.email johndoe@example.com

提示：
    windows配置文件位置：C:\Users\[Administrator]\.gitconfig

二、项目与GitHub交互

1. 本地项目上传到GitHub仓库
2. 从GitHub仓库克隆项目

2.1 本地项目上传到GitHub仓库

git init # 初始化项目                            --->生成Git所需配置文件

git add . # 把当前目录下的所有文件，交给git管理 --->把文件添加到暂存区

git commit -m "first commit" # 提交--->把暂存区文件提交到本地仓库

git remote add origin https://github.com/LGY2008/test02.git # 建立远端连接 

git push -u origin master # 推送到远端            --->把本地仓库内容推送到远端仓库

2.2 从GitHub仓库克隆项目

1. 首次克隆项目 git clone https://github.com/LGY2008/UIAUTO.git
2. 从GitHub获取最新代码 git pull

三、冲突

在多个人开发中，多人对同一个文件同一行代码的修改。在pull的时候，会发生。
（当远端的某一个文件的某一行。和本地不同， 并且不是之前的版本）
<<<<<<< HEAD
        allure.attach("描述，打印第二句话","")
        print("test003被执行")
=======
>>>>>>> e70a1a33f9c903ae7cc4a145e93e4c1e1bbb6a85

HEAD部分是自己的代码
远端版本号的代码是远端的代码

解决的问题的方法，需要自己权衡，保留或者删除删(多余的<<<< ===== >>>> )相关信息;

重新提交，并推送。

四、git操作方式

1. 命令
2. 工具（
    1. 第三方工具 如：Sourcetree
    2. Git工具
    3. 开发工具 Pycharm

提示：无论用哪款工具，后台调用的都是git命令

4.1 命令 【了解】

0. 创建本地git仓库  git init
1. 查看状态 git status
2. 把文件添加到暂存区 git add .
3. 把暂存区命令提交到git本地仓库 git commit -m "注释"
4. 把本地仓库绑定到远程仓库(注意：初次在本地创建仓库时使用) 
   (命令：git remote add origin https://github.com/LGY2008/TestSMS001.git)
5. 克隆 git clone 
6. 拉取：git pull
